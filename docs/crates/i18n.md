# i18n (`crates/ui/src/strings.rs`)

Minimal internationalization infrastructure: flat/interpolated string lookup keyed by a
flat dotted key (e.g. `"directory.search_hint"`), backed by an embedded per-locale JSON
file (`assets/locales/<code>.json`). English-only today — this is infrastructure for a
second locale, not a translation itself.

## Architecture

`init(language: Language)` parses and caches the matching locale JSON into a
process-wide `OnceLock<HashMap<String, String>>`, called once at `DeelipApp::new`
before any lookup. `t(key) -> String` looks up a flat string; `tf(key, args) -> String`
does the same lookup then substitutes `{name}`-style placeholders from `args` (e.g.
`tf("dialer.calling", &[("name", &caller_name)])` against a locale string like
`"Calling {name}…"`). `deelip_config::Language` (currently a single `En` variant) is the
`AppConfig` field driving which JSON file loads.

## Design decisions & invariants

**Fails visible, not silent or panicking**: a lookup before `init` has run, or a key
missing from the parsed file, returns the raw key itself rather than a blank string or
a panic — an unmigrated call site or a genuine typo is obvious at a glance during
verification instead of silently rendering nothing.

**Icon-glyph-plus-label composites** (e.g. `"{icon}  Complete Transfer"`) keep the icon
concatenation in Rust and pass only the label half through `t`/`tf` — glyphs aren't
translatable text, and baking one into a locale string would make the JSON file
font-stack-dependent.

**Pluralization is out of scope for now**: the English singular/plural branch (e.g. the
tray tooltip's missed-call count) stays in Rust as an explicit `if`, with each branch's
fixed text as its own separate locale key, rather than any generic plural-rule
machinery.

## Known limitations / open items

- No second translated locale exists yet — this is the load-bearing infrastructure
  piece, not a completed migration to a second language.
- RTL layout support is explicitly out of scope.
- Migration to `t`/`tf` across the ~216 original call sites was done view-by-view,
  starting with smaller views to validate the lookup API's shape before the larger
  ones (`settings.rs`/`dialer.rs`) — by the time of this doc, migration is complete
  across the crate.
